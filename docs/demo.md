# Демонстрация проекта

## Подготовка

Для запуска сохранённой модели нужен Python 3.12.x:

```bash
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[app,model-runtime]"
streamlit run app.py
```

На Windows окружение активируется командой `.venv\Scripts\activate`.

## Сценарий на 5 минут

1. В разделе **«Обзор проекта»** покажите размер данных: 801 образец, 20 531 признак и
   пять классов опухолевой ткани.
2. В разделе **«Эксперименты»** объясните, что preprocessing находится внутри Pipeline,
   модели сравнивались только на train-части, а E10 был выбран до test-оценки.
3. Покажите устойчивость E10 на 50 train-only validation folds.
4. В разделе **«Финальный результат»** покажите confusion matrix и интервал Wilson. Метрика
   1.0 относится к 161 объекту одного фиксированного test split.
5. В разделе **«Smoke-проверка»** загрузите `examples/demo_sample.csv`. Ожидаемый класс —
   `KIRC`. Пример взят из уже оценённого фиксированного test split и используется только для
   технической демонстрации загрузки Pipeline.

Если запуск Streamlit невозможен, откройте на GitHub
`notebooks/model_comparison_demo.ipynb`: таблицы, графики и outputs уже сохранены.

## Что важно сказать

- Задача проекта — различить BRCA, COAD, KIRC, LUAD и PRAD по подготовленной матрице UCI.
- Основная метрика — macro F1, потому что классы несбалансированы.
- `StandardScaler`, PCA и SelectKBest обучаются отдельно внутри каждого fold, поэтому
  preprocessing не видит validation-часть.
- Test set был зафиксирован и использован после выбора E10.
- Результат не является медицинской диагностикой и не доказывает переносимость на данные
  другой лаборатории.

## Ответы на ожидаемые вопросы

**Почему качество равно 1.0?**  Пять типов опухолевой ткани хорошо разделяются в этой
внутренней когорте. При этом нижняя граница 95% интервала Wilson для accuracy равна 0.9767,
а внешняя когорта не исследовалась.

**Есть ли утечка данных?**  Преобразования включены в sklearn Pipeline и обучаются внутри
каждого fold. Отдельный test split не использовался при сравнении моделей.

**Почему нет внешней проверки?**  Сопоставимой независимой когорты в проекте нет.
Синтетические данные не заменили бы реальную внешнюю валидацию, поэтому это ограничение
указано явно.
