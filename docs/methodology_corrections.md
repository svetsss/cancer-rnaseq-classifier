# Методологические исправления после ревью

Финальные test-артефакты и история их получения сохраняются без изменений. Исправления ниже
относятся к новым train CV-запускам и сохраняют зафиксированную test-оценку.

## Выбор кандидата

Время выполнения и прямое сравнение числа PCA-компонент с количеством выбранных признаков
удалены из tie-break. Текущий код учитывает среднее и стандартное отклонение macro F1,
семейство классификатора и метод представления. Размерность сравнивается после выбора метода.

## MLP

Сохранённая строка E17 была получена протоколом, в котором модель после early stopping
оставалась обученной только на inner-train. Текущий код использует inner-validation для
выбора числа эпох, затем заново обучает selector, scaler и MLP на полном outer-train.

Исправленный MLP-протокол повторно выполнен на 640 train-объектах командой
`python -m scripts.run_mlp_training_analysis --publish`. Получен CV macro F1
`0.998699 ± 0.002603`.

Новые артефакты:

- `results/mlp_training_analysis.json` — история loss и macro F1 по эпохам;
- `results/mlp_fold_metrics.csv` — метрики и выбранные эпохи по folds;
- четыре отдельных Matplotlib-графика в `figures/mlp_*.png`.

Строка E17 в `results/metrics.csv` заменена результатом исправленного протокола. Это единый
агрегированный источник для README, Streamlit и графиков: macro F1
`0.998699 ± 0.002603`. Детализация по folds находится в
`results/mlp_fold_metrics.csv`, а история эпох — в `results/mlp_training_analysis.json`.
Результат E10 и замороженная test-оценка при этом не менялись.

## Устойчивость

`python -m scripts.run_robustness_analysis --repeats 10 --publish` выполнил 50 train-only
оценок E10. Получен macro F1 `0.998040 ± 0.004493`, диапазон
`0.987349–1.000000`. Результат сохранён в `results/robustness_e10.json`, отдельный график —
в `figures/e10_robustness.png`. Замороженные test-артефакты не изменялись.
