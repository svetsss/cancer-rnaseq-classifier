# Классификация типов опухолей по данным RNA-Seq

Учебный проект по анализу высокоразмерных данных экспрессии генов. Реализованы первичная
проверка датасета UCI Gene Expression Cancer RNA-Seq, воспроизводимое разделение данных и
baseline-модели, а также fold-local отбор признаков. Проект не является медицинской
диагностической системой. Исследовательское сравнение дополнено PCA, Random Forest и
небольшой нейронной сетью на PyTorch.

## Что реализовано

- загрузка исходного архива UCI без изменения его содержимого;
- разделение матрицы признаков и целевой переменной;
- проверка структуры, числовых типов, пропусков, бесконечных значений и дубликатов;
- подсчёт константных признаков без их удаления;
- сохранение фактической сводки датасета в JSON;
- построение распределения классов;
- стратифицированное разделение на train и test;
- 5-fold cross-validation на train-части;
- DummyClassifier и Logistic Regression со стандартизацией внутри Pipeline;
- сравнение Logistic Regression и LinearSVC после SelectKBest;
- сравнение Logistic Regression и LinearSVC после PCA;
- Random Forest после SelectKBest;
- MLP с внутренней validation-частью для early stopping в каждом внешнем fold;
- сохранение CV-метрик и воспроизводимого описания разделения;
- однократное обучение и test-оценка заранее зафиксированного E10;
- сохранение итогового Pipeline, predictions, classification report и confusion matrix;
- офлайн-тесты на малом локальном архиве.

## Анализ датасета

Команда загружает официальный архив UCI в `data/raw/`, проверяет данные и создаёт два
артефакта:

```bash
python -m scripts.analyze_dataset
```

- `results/dataset_summary.json` — размерность, контроль качества и распределение классов;
- `figures/class_distribution.png` — столбчатая диаграмма числа объектов каждого класса.

Повторный запуск использует локальный архив, только если он открывается как tar.gz и содержит
оба CSV-файла датасета. Повреждённый или неполный файл загружается заново.

## Baseline-модели

Команда создаёт стратифицированное разделение 80/20 с `random_state=42`, резервирует test
и выполняет 5-fold `StratifiedKFold` только на train-части:

```bash
python -m scripts.run_baselines
```

- `results/split_summary.json` — размеры, распределения классов, параметры CV и SHA-256
  последовательностей идентификаторов объектов в train и test;
- `results/metrics.csv` — CV-метрики E00 и E01, число признаков и время выполнения.

В train вошло 640 объектов, в test — 161. Распределение классов в train: BRCA — 240,
COAD — 62, KIRC — 116, LUAD — 113, PRAD — 109; в test: BRCA — 60, COAD — 16,
KIRC — 30, LUAD — 28, PRAD — 27.

Обе модели используют все 20 531 признаков. E00 — `DummyClassifier` со стратегией
`most_frequent`. E01 — `StandardScaler` и `LogisticRegression(max_iter=3000)` внутри
одного `Pipeline`.

| Эксперимент | CV macro F1 | CV accuracy | CV precision macro | CV recall macro | Полное время CV, с |
|---|---:|---:|---:|---:|---:|
| E00 DummyClassifier | 0.109091 | 0.375000 | 0.075000 | 0.200000 | 0.83 |
| E01 LogisticRegression | 0.995388 | 0.995313 | 0.993517 | 0.997500 | 2.29 |

Это оценки cross-validation на train-части. Test set не участвовал в preprocessing,
обучении, выборе конфигураций или расчёте метрик и остаётся зарезервированным для финальной
оценки на последующем этапе.

## Расширенное сравнение моделей

Команда воспроизводит сохранённое разделение, не используя test set для экспериментов или
оценки, и выполняет E10-E17:

```bash
python -m scripts.run_extended_experiments
```

В E10-E15 используются Pipeline `StandardScaler` → `PCA(svd_solver="randomized")` →
classifier. E16 использует `SelectKBest(f_classif, k=200)` и
`RandomForestClassifier(n_estimators=300)`. Для E17 внешний 5-fold CV отделён от внутренней
validation-части: selector и scaler обучаются только на inner-train, early stopping смотрит
только на inner-validation, а outer-validation оценивается один раз.

| ID | Модель | Представление | CV macro F1, mean ± std | CV accuracy | CV precision macro | CV recall macro | Время CV, с |
|---|---|---:|---:|---:|---:|---:|---:|
| E10 | LogisticRegression | PCA 20 | 1.000000 ± 0.000000 | 1.000000 | 1.000000 | 1.000000 | 2.33 |
| E11 | LogisticRegression | PCA 50 | 0.998657 ± 0.002685 | 0.998437 | 0.999184 | 0.998182 | 2.50 |
| E12 | LogisticRegression | PCA 100 | 0.998657 ± 0.002685 | 0.998437 | 0.999184 | 0.998182 | 2.64 |
| E13 | LinearSVC | PCA 20 | 0.998728 ± 0.002544 | 0.998437 | 0.998333 | 0.999167 | 2.35 |
| E14 | LinearSVC | PCA 50 | 0.988925 ± 0.012215 | 0.992188 | 0.986066 | 0.993116 | 2.55 |
| E15 | LinearSVC | PCA 100 | 0.991222 ± 0.008227 | 0.993750 | 0.988352 | 0.994855 | 2.67 |
| E16 | RandomForestClassifier | SelectKBest 200 | 0.996055 ± 0.003222 | 0.995313 | 0.997551 | 0.994704 | 2.83 |
| E17 | PyTorch MLP | SelectKBest 200 | 0.997356 ± 0.003239 | 0.996875 | 0.998367 | 0.996443 | 4.00 |

E10 и E08 достигли одинакового максимального CV macro F1. E10 рассматривается как
кандидат для финальной оценки: среди лидеров у него наименьшая размерность внутреннего
представления — 20 компонент против 200 и 500 выбранных признаков у E08 и E09. Компоненты
PCA являются плотными комбинациями исходных признаков, поэтому это не означает использование
только 20 генов. E16 и E17 не улучшили результат E08. В E17 ранняя остановка завершила
folds за 29, 26, 14, 14 и 21 эпоху (в среднем 20.8).

Предварительное сравнение по train CV:

- лучшая SelectKBest-конфигурация — E08;
- лучшая PCA-конфигурация и лучший классический алгоритм по правилам сравнения — E10;
- Random Forest E16 — 0.996055 macro F1;
- PyTorch MLP E17 — 0.997356 macro F1;
- максимальный результат 1.0 получили E08, E09 и E10;
- конфигурация с наименьшей размерностью внутреннего представления среди лидеров — E10.

Созданы [сводка обучения MLP](results/mlp_summary.json),
[сравнение PCA](figures/pca_comparison.png),
[сравнение моделей](figures/model_comparison.png) и
[двумерная PCA-проекция train](figures/pca_train_projection.png). Проекция является

## Зафиксированный кандидат для финальной оценки

До обращения к test set выбранная по train CV конфигурация зафиксирована командой:

```bash
python -m scripts.freeze_final_candidate
```

Выбран E10 со следующим Pipeline: `StandardScaler` →
`PCA(n_components=20, svd_solver="randomized", random_state=42, whiten=False)` →
`LogisticRegression(max_iter=3000, random_state=42)`. E08, E09 и E10 имели одинаковый CV
macro F1 `1.0 ± 0.0`. Tie-break последовательно учитывал максимальный macro F1, меньшую
размерность внутреннего представления, меньшее время и более простую модель.

Двадцать PCA-компонент — это размерность внутреннего представления. PCA по-прежнему
использует все 20 531 исходный признак; результат нельзя описывать как модель на 20 генах.
Решение принято без загрузки датасета, только по сохранённым `results/metrics.csv` и
`results/split_summary.json`. На момент фиксации test set ещё не оценивался, модель на полном
train не обучалась. Запись находится в `results/final_candidate.json`. После фиксации менять
кандидата нельзя без нарушения протокола.
